# Thyme OS Brand Guidelines

## Brand Identity

**Mission**: Making Linux accessible on vintage MacBook hardware through specialized compatibility solutions.

**Values**: 
- Sustainability (extending hardware life)
- Accessibility (removing technical barriers)
- Community-driven development
- Hardware preservation

## Visual Identity

### Logo System

#### Primary Logo (`logo.svg`)
- **Usage**: Headers, marketing materials, large formats
- **Minimum size**: 60px width
- **Background**: Use on light backgrounds only

#### Icon/Favicon (`favicon.svg`)
- **Usage**: Browser tabs, app icons, small formats
- **Size**: 32x32px optimized
- **Scalable**: Works at 16px minimum

#### Wordmark (`wordmark.svg`)
- **Usage**: Headers, documentation, professional materials
- **Includes**: Full "THYME Operating System" text
- **Tagline**: "Linux for MacBooks"

### Color Palette

#### Primary Colors
- **Thyme Green**: `#2E7D32` (Primary brand color)
- **Deep Green**: `#1B5E20` (Accent/borders)
- **Light Green**: `#66BB6A` (Highlights)
- **Sage Green**: `#81C784` (Secondary elements)
- **Pale Green**: `#A5D6A7` (Subtle accents)

#### Supporting Colors
- **Stem Brown**: `#4E342E` (Natural element)
- **White**: `#FFFFFF` (Text on colored backgrounds)
- **Charcoal**: `#2C2C2C` (Body text)

### Typography

#### Headers
- **Font**: Arial, Helvetica, sans-serif
- **Weight**: Bold (700)
- **Usage**: Main headings, navigation

#### Body Text
- **Font**: Arial, Helvetica, sans-serif
- **Weight**: Regular (400)
- **Usage**: Content, descriptions

#### Technical Text
- **Font**: Consolas, Monaco, monospace
- **Weight**: Regular (400)
- **Usage**: Code, terminal commands

### Logo Symbolism

#### Thyme Herb
- **Represents**: Natural, organic approach to technology
- **Connection**: "Thyme" wordplay with "Time" - extending hardware lifetime
- **Visual**: Stylized herb sprig with leaves

#### Apple Bite Reference
- **Subtle nod**: To MacBook/Apple hardware compatibility
- **Positioning**: Small, not dominant
- **Message**: We work with Apple hardware, not against it

#### Tech Elements
- **Small squares**: Represent digital/technical aspects
- **Scattered placement**: Integration of nature and technology
- **Color**: Light green for subtlety

## Brand Voice

### Tone
- **Friendly but knowledgeable**
- **Technical without being intimidating**
- **Encouraging for vintage hardware users**
- **Community-focused**

### Language
- Use "vintage" not "old" for hardware
- "Revive" not "fix" MacBooks
- "Community-driven" not "free"
- "Compatibility layer" not "hack"

### Messaging Examples
- ✅ "Revive your vintage MacBook with modern Linux"
- ✅ "Purpose-built for MacBook hardware compatibility"
- ❌ "Fix your broken old MacBook"
- ❌ "Hack around Apple's limitations"

## Usage Guidelines

### Do
- Use official logo files
- Maintain minimum sizes
- Preserve logo proportions
- Use approved color combinations
- Keep adequate whitespace around logos

### Don't
- Recreate or modify logos
- Use colors outside the brand palette
- Place logos on busy backgrounds
- Stretch or distort brand elements
- Use outdated versions

### File Formats

#### Vector Files
- **SVG**: Web use, scalable
- **AI/EPS**: Print production (when available)

#### Raster Files
- **PNG**: Web with transparency
- **JPG**: Photography backgrounds only

## Application Examples

### Website Headers
```html
<img src="branding/wordmark.svg" alt="Thyme OS - Linux for MacBooks" width="200">
```

### Favicon
```html
<link rel="icon" type="image/svg+xml" href="branding/favicon.svg">
```

### Social Media Profile
- Use primary logo on white/light green background
- Include tagline: "Linux for MacBooks"

## Brand Extensions

### Hardware Compatibility Badges
- "Thyme OS Compatible" badges for tested hardware
- Green check mark with thyme icon
- Use sage green (#81C784) for compatibility indicators

### Community Elements
- Forum avatars use simplified thyme icon
- Documentation headers use wordmark
- GitHub repository uses primary logo
